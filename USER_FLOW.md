# SpotSnap 사용자 흐름 (User Flow)

## 📱 전체 아키텍처

### 화면 구조
```
로그인/회원가입 (/)
    ↓
온보딩 - 취향 설정 (/onboarding)
    ↓
메인 서비스 ←→ 하단 네비게이션
    ├── 지도 (/main)
    ├── 스팟 리스트 (/spots)
    ├── 커뮤니티 (/community)
    └── 프로필 (/profile)
```

---

## 🎯 주요 사용자 시나리오

### 1️⃣ 신규 사용자 온보딩

**시나리오:** 처음 앱을 설치한 사용자가 서비스를 시작합니다.

```
1. 로그인/회원가입 (/)
   ├─ 이메일/비밀번호 입력
   ├─ 소셜 로그인 (카카오/네이버/구글)
   └─ 회원가입 시 → 2단계로 이동
   
2. 온보딩 - 취향 설정 (/onboarding)
   ├─ Step 1: 배경 유형 선택 (자연/유적지/건물/카페/바다 등)
   ├─ Step 2: 시간대 선택 (주간/야간/노을/새벽)
   ├─ Step 3: 사진 스타일 선택 (감성/현실/몽환/필름 등)
   └─ 완료 → 메인 지도로 이동
   
3. 메인 지도 화면 (/main)
   └─ 개인화된 추천 스팟 표시
```

**특징:**
- 건너뛰기 가능
- 언제든지 프로필에서 재설정 가능
- 3단계로 간단하게 구성

---

### 2️⃣ 촬영 장소 찾기 (위치 기반)

**시나리오:** 주말에 사진 찍을 장소를 찾는 사용자

```
1. 메인 지도 화면 (/main)
   ├─ 현재 위치 중심으로 지도 표시
   ├─ 주변 촬영 스팟 마커 표시
   │  ├─ 미방문: 흰색 마커
   │  └─ 방문 완료: 핑크색 마커 + 반투명 영역
   ├─ 필터 적용
   │  ├─ 카테고리 (자연/카페/바다 등)
   │  ├─ 거리 (1km/5km/10km)
   │  └─ 평점 (4.0 이상 등)
   └─ 검색 (지역명, 장소명)

2. 마커 클릭
   ├─ 하단에 카드 팝업
   │  ├─ 장소 사진
   │  ├─ 이름, 주소
   │  ├─ 평점, 리뷰 수
   │  ├─ 거리, 이동 시간
   │  ├─ 태그
   │  └─ [자세히 보기] 버튼
   └─ 자세히 보기 클릭 → 3단계로 이동

3. 장소 상세 페이지 (/spot/:id)
   ├─ Tab: 정보
   │  ├─ 소개
   │  ├─ 카테고리
   │  ├─ 추천 시간대
   │  ├─ 촬영 팁
   │  ├─ 태그
   │  └─ 이동 정보 + [길찾기] 버튼
   ├─ Tab: 사진
   │  ├─ 사용자가 업로드한 사진 갤러리
   │  └─ [사진 올리기] 버튼 → 업로드 페이지로
   └─ Tab: 리뷰
       ├─ 별점 및 텍스트 리뷰
       └─ [리뷰 작성] 버튼

4. 액션
   ├─ 북마크 추가/제거
   ├─ 방문 완료 체크
   ├─ 공유하기
   └─ 길찾기 (외부 지도 앱 연동)
```

**특징:**
- 실시간 위치 기반 추천
- 시각적 지도 인터페이스
- 방문 이력 시각화

---

### 3️⃣ 스팟 리스트 탐색

**시나리오:** 리스트 형태로 여러 스팟을 비교하고 싶은 사용자

```
1. 스팟 리스트 (/spots)
   ├─ Tab: 전체 / 북마크 / 내 주변
   ├─ 정렬 옵션
   │  ├─ 거리순
   │  ├─ 평점순
   │  ├─ 리뷰순
   │  └─ 최신순
   ├─ 카드 형태로 스팟 표시
   │  ├─ 대표 이미지
   │  ├─ 이름, 설명
   │  ├─ 평점, 리뷰 수
   │  ├─ 거리, 이동 시간
   │  ├─ 태그
   │  └─ 북마크 버튼
   └─ 카드 클릭 → 장소 상세 페이지

2. 장소 상세 페이지 (/spot/:id)
   └─ (위 "2️⃣ 촬영 장소 찾기"의 3단계와 동일)
```

**특징:**
- 한눈에 여러 스팟 비교
- 카드형 인스타그램 스타일 UI
- 빠른 필터링 및 정렬

---

### 4️⃣ 사진 업로드 및 공유

**시나리오:** 촬영한 사진을 커뮤니티에 공유하는 사용자

```
1. 진입 경로
   ├─ 커뮤니티 페이지 → [새 게시글 작성] 버튼
   └─ 장소 상세 페이지 → 사진 탭 → [사진 올리기] 버튼

2. 사진 업로드 (/upload)
   ├─ 사진 선택 (최대 10장)
   │  ├─ 갤러리에서 선택
   │  ├─ 드래그 앤 드롭
   │  └─ 대표 사진 선택
   ├─ 촬영 장소 선택 (필수)
   │  └─ 드롭다운으로 스팟 검색
   ├─ 설명 작성 (최대 1000자)
   ├─ 태그 추가 (선택)
   │  └─ #태그 형식으로 입력
   └─ [게시하기] 버튼

3. 게시 완료
   └─ 커뮤니티 페이지로 이동
```

**특징:**
- 인스타그램 스타일 업로드 플로우
- 반드시 장소와 연결
- 사진 중심 콘텐츠

---

### 5️⃣ 커뮤니티 활동

**시나리오:** 다른 사용자의 사진을 보고 소통하는 사용자

```
1. 커뮤니티 (/community)
   ├─ Tab: 게시글
   │  ├─ 피드 스타일 게시글 목록
   │  │  ├─ 프로필 사진, 이름
   │  │  ├─ 장소 링크 (클릭 시 장소 상세로)
   │  │  ├─ 사진 (스와이프 가능)
   │  │  ├─ 설명 텍스트
   │  │  └─ 좋아요, 댓글, 북마크, 공유
   │  └─ 스크롤로 무한 로딩
   │
   └─ Tab: 모임
       ├─ 모임 카드 목록
       │  ├─ 모임 제목
       │  ├─ 설명
       │  ├─ 일시, 시간, 장소
       │  ├─ 참여 인원 (3/8명)
       │  └─ [참여하기] 버튼
       └─ [새 모임 만들기] 버튼

2. 게시글 상세 (구현 예정)
   ├─ 전체 사진 및 설명 보기
   ├─ 댓글 목록
   │  └─ 대댓글 지원
   └─ 댓글 작성

3. 모임 상세 (구현 예정)
   ├─ 모임 정보
   ├─ 참여자 목록
   ├─ 그룹 채팅
   └─ [참여하기] / [참여 취소]
```

**특징:**
- 인스타그램 스타일 피드
- 장소 중심 커뮤니티
- 오프라인 모임 기능

---

### 6️⃣ 채팅

**시나리오:** 다른 사용자와 1:1 또는 그룹 채팅

```
1. 채팅 목록 (/chat)
   ├─ 채팅방 리스트
   │  ├─ 프로필 사진
   │  ├─ 이름 (1:1) 또는 모임명 (그룹)
   │  ├─ 마지막 메시지
   │  ├─ 시간
   │  └─ 읽지 않은 메시지 수
   └─ 채팅방 클릭 → 2단계로

2. 채팅방 (/chat?room=:id)
   ├─ 메시지 목록
   │  ├─ 상대방 메시지 (왼쪽 정렬)
   │  └─ 내 메시지 (오른쪽 정렬, 핑크색)
   ├─ 입력창
   │  ├─ 이미지 첨부
   │  ├─ 이모지
   │  └─ 메시지 전송
   └─ 실시간 알림 (SSE 기반)
```

**특징:**
- 실시간 메시징
- 1:1 및 그룹 채팅
- 모임과 연동

---

### 7️⃣ 마이페이지 및 기록 관리

**시나리오:** 내 활동 내역을 확인하는 사용자

```
1. 프로필 (/profile)
   ├─ 프로필 정보
   │  ├─ 프로필 사진
   │  ├─ 이름, 소개
   │  └─ [프로필 수정] 버튼
   ├─ 통계
   │  ├─ 방문한 스팟 (숫자)
   │  ├─ 북마크 (숫자)
   │  └─ 사진 (숫자)
   ├─ 내 취향 (카테고리 태그)
   └─ 메뉴
       ├─ 방문한 스팟 → 2단계로
       ├─ 북마크 → 3단계로
       ├─ 내 사진첩
       ├─ 내 모임
       ├─ 채팅
       ├─ 설정
       └─ 로그아웃

2. 방문한 스팟 (/visited)
   ├─ 방문 완료한 스팟 목록
   ├─ 카드 형태로 표시
   └─ 카드 클릭 → 장소 상세

3. 북마크 (/bookmarks)
   ├─ 북마크한 스팟 목록
   ├─ 카드 형태로 표시
   └─ 카드 클릭 → 장소 상세

4. 내 사진첩 (구현 예정)
   ├─ 업로드한 사진 그리드
   └─ 사진 클릭 → 게시글 상세

5. 내 모임 (구현 예정)
   ├─ 참여 중인 모임 목록
   └─ 모임 클릭 → 모임 상세
```

**특징:**
- 개인 활동 통계
- 방문 이력 관리
- 북마크 컬렉션

---

### 8️⃣ 관리자 기능 (구현 예정)

**시나리오:** 관리자가 콘텐츠를 관리합니다.

```
1. 관리자 대시보드 (/admin)
   ├─ 통계
   │  ├─ 총 사용자 수
   │  ├─ 총 스팟 수
   │  ├─ 총 게시글 수
   │  └─ 일일 활성 사용자
   ├─ 승인 대기 목록
   │  ├─ 사용자 제안 장소
   │  └─ [승인] / [거절] 버튼
   └─ 신고된 콘텐츠
       ├─ 부적절한 게시글
       └─ [삭제] / [무시] 버튼
```

## 📱 반응형 디자인

### 모바일 (< 1024px)
- 하단 네비게이션 표시
- 전체 화면 사용
- 단일 컬럼 레이아웃
- 터치 최적화

### 웹/태블릿 (≥ 1024px)
- 하단 네비게이션 숨김
- 최대 너비 제한 (max-w-7xl)
- 그리드 레이아웃 (2-3 컬럼)
- 마우스 호버 효과

## 🔄 주요 상호작용

### 지도
- 마커 클릭 → 카드 팝업
- 필터 적용 → 마커 업데이트
- 검색 → 지도 이동

### 리스트
- 카드 클릭 → 상세 페이지
- 북마크 버튼 → 즉시 저장
- 정렬 변경 → 리스트 재정렬

### 상세 페이지
- 탭 전환 → 콘텐츠 변경
- 북마크/방문 완료 → 상태 토글
- 길찾기 → 외부 앱 연동

### 커뮤니티
- 좋아요 → 하트 애니메이션
- 댓글 → 카운트 증가
- 모임 참여 → 버튼 상태 변경

## 📝 기술 스택 요약

- **Frontend**: React + TypeScript + Tailwind CSS
- **Routing**: React Router (Data Mode)
- **Animation**: Motion (Framer Motion)
- **Icons**: Lucide React
- **UI Components**: Radix UI
- **State**: React Hooks (로컬 상태)
- **Backend** (향후): Supabase
  - Authentication
  - Database (PostgreSQL)
  - Storage (이미지)
  - Realtime (채팅, 알림)
  - Edge Functions (추천 알고리즘)

이 User Flow는 사진 촬영 장소 추천 플랫폼의 전체적인 사용자 경험을 설명합니다. 
각 단계는 직관적이고 간결하게 설계되어 있으며, 모바일과 웹 모두에서 최적화된 경험을 제공합니다.
